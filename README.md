# Music Store SQL Data Analysis

## Project Overview
This project analyzes a **Music Store relational database using SQL** to uncover insights about customer behavior, sales performance, and artist popularity. The analysis answers key business questions related to revenue generation, customer spending patterns, and music genre preferences.

---

## Tools & Technologies
- SQL  
- PostgreSQL  
- Data Analysis  
- Relational Database Queries  

---

## Business Questions Solved
1. Identify the **senior-most employee** based on job title.  
2. Determine **countries generating the highest number of invoices**.  
3. Find the **top 3 highest invoice totals**.  
4. Identify the **city generating the highest revenue**.  
5. Determine the **highest spending customer**.  
6. Retrieve **Rock music listeners and their contact details**.  
7. Identify the **top 10 artists producing the most Rock tracks**.  
8. Find **songs longer than the average track length**.  

---

## SQL Concepts Used
- Joins  
- Aggregations (`SUM`, `COUNT`, `AVG`)  
- Subqueries  
- `GROUP BY` operations  
- Sorting & Filtering (`ORDER BY`, `LIMIT`)  
- Data Exploration  

---

## Dataset
The project uses a **Music Store relational database** containing tables such as:

- Customers  
- Invoices  
- Invoice Lines  
- Tracks  
- Albums  
- Artists  
- Genres  
- Employees  

---

## Project Outcome
This analysis demonstrates how SQL can be used to **extract meaningful business insights from relational databases**, helping organizations understand **customer behavior, revenue trends, and product popularity** through data-driven analysis.
